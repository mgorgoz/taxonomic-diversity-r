# Calculate Ozkan's pTO Using the Original Excel Macro Formula

Computes pTO using the weighting scheme from the original Ozkan Excel
macro (TD_OMD). This differs from
[`ozkan_pto()`](https://mgorgoz.github.io/taxonomic-diversity-r/reference/ozkan_pto.md)
in how slicing steps are weighted: the macro uses only the number of
"active" steps (steps where \>= 2 species survive) rather than n_s =
max_abundance as described in the published paper.

## Usage

``` r
ozkan_pto_macro(community, tax_tree)
```

## Arguments

- community:

  A named numeric vector of species abundances. Names must match the
  first column of `tax_tree`.

- tax_tree:

  A data frame with taxonomic hierarchy. First column is species names,
  subsequent columns are taxonomic ranks from lowest to highest (e.g.,
  Species, Genus, Family, Order, Class, Phylum, Kingdom).

## Value

A named list with components:

- uTO:

  Unweighted taxonomic diversity (macro formula)

- TO:

  Weighted taxonomic diversity (macro formula)

- uTO_plus:

  Unweighted taxonomic distance (same as paper)

- TO_plus:

  Weighted taxonomic distance (same as paper)

- Ed_levels:

  Deng entropy at each taxonomic level (nk=0 slice)

## Details

The Excel macro (TD_OMD, Ozkan 2018a) counts the number of slicing steps
where the Deng entropy product is non-zero (`n_active`), then assigns
weights `n_active, n_active-1, ..., 1` to those steps only. Steps where
1 or fewer species survive are excluded entirely.

In contrast,
[`ozkan_pto()`](https://mgorgoz.github.io/taxonomic-diversity-r/reference/ozkan_pto.md)
follows the published formula (Ozkan 2018, Eq. 4) which uses
`n_s = max_abundance` for all steps, including single-species steps
(with product = 1 by the postulation rule).

The two functions give identical results when all slicing steps have 2
or more surviving species (`n_active == n_s`). They differ when some
steps reduce to 1 or fewer species, which typically occurs in
species-poor or abundance-skewed communities.

This function is provided for backward compatibility with results
generated by the original Excel macro. For new analyses, use
[`ozkan_pto()`](https://mgorgoz.github.io/taxonomic-diversity-r/reference/ozkan_pto.md)
which implements the published formula.

## Note

Maximum 9 slicing steps, matching the Westhoff & van der Maarel (1973)
abundance scale used in the original macro.

## References

Ozkan, K. (2018). A new proposed measure for estimating taxonomic
diversity. Turkish Journal of Forestry, 19(4), 336-346. DOI:
10.18182/tjf.441061

## See also

[`ozkan_pto()`](https://mgorgoz.github.io/taxonomic-diversity-r/reference/ozkan_pto.md)
for the published paper formula,
[`deng_entropy_level()`](https://mgorgoz.github.io/taxonomic-diversity-r/reference/deng_entropy_level.md)
for the core Deng entropy calculation.

## Examples

``` r
comm <- c(sp1 = 4, sp2 = 2, sp3 = 3, sp4 = 1, sp5 = 2)
tax <- data.frame(
  Species = paste0("sp", 1:5),
  Genus   = c("G1", "G1", "G1", "G2", "G2"),
  Family  = c("F1", "F1", "F1", "F1", "F1"),
  stringsAsFactors = FALSE
)
ozkan_pto_macro(comm, tax)
#> $uTO
#> [1] 2.795401
#> 
#> $TO
#> [1] 3.473158
#> 
#> $uTO_plus
#> [1] 3.060705
#> 
#> $TO_plus
#> [1] 3.753852
#> 
#> $Ed_levels
#>  Species    Genus   Family 
#> 1.609438 2.280003 0.000000 
#> 
```
